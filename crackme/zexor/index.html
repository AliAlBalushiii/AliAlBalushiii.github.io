Target Overview

ZEXOR v0.1 is a basic license checker designed for beginners to practice reverse engineering skills. The program prompts for a license key and validates it against a hardcoded value.

  • Name: ZEXOR v0.1
  • Author: svidnet
  • Platform: Windows x86-64

  • Type: Crackme / License Checker

Initial Analysis

When executing the binary, we're immediately presented with a console application requesting a license key:

Figure 1: Initial execution of ZEXOR v0.1

The application provides no feedback on incorrect keys, making static analysis the preferred approach for understanding the validation mechanism.

Analysis Approach

Since this is a beginner-level crackme, we can expect straightforward string comparison logic without obfuscation or anti-debugging techniques. The focus will be on locating and understanding the validation routine.

Static Analysis with IDA Pro

Locating the CheckLicense() Function

Opening the executable in IDA Pro and examining the function window reveals a promising function named CheckLicense():

Figure 2: CheckLicense() function in IDA Pro

Examining the decompiled code shows a clear strcmp() call:

C 
int CheckLicense() {
    char user_input[256];
    // ... get user input ...
    int result = strcmp(user_input, expected_license);
    return result;
}
x64 Calling Convention Note

On x64 Windows systems, the strcmp() function follows the Microsoft x64 calling convention:
RCX = First parameter (user input)
RDX = Second parameter (expected license key)

Analyzing the strcmp() Parameters

Our focus shifts to understanding what's being passed as the second parameter to strcmp() (the expected license key in RDX register).

Figure 3: RDX assigned to cs:LICENO

The assembly shows RDX is assigned to cs:LICENO. The cs: prefix indicates that LICENO is a global variable stored in the code segment.

Following the LICENO Pointer

Navigating to the LICENO symbol reveals it's a pointer:

Figure 4: LICENO is a pointer (dq = quadword on x64)

In x64 architecture, dq (define quadword) represents an 8-byte address. LICENO is a pointer that holds the address of the actual license string.

Key Insight

The license key isn't stored directly at LICENO. Instead, LICENO contains a memory address that points to the actual license string. We need to dereference this pointer to find the key.

Extracting the License Key

Following the pointer value stored at LICENO leads us to the actual license string:

Figure 5: Memory location containing the license key
Figure 6: The extracted license key

License Key Found: ZEXOR-2025-SVIDNET-CRACKME

Testing this key in the application confirms successful validation:

License Validated Successfully!

Technical Summary

  • The crackme uses a simple strcmp() comparison for license validation
  • The expected license is stored as a global pointer (LICENO) in the code segment
  • No obfuscation, encryption, or anti-debugging techniques are employed
  • This is an ideal beginner exercise for understanding x64 calling conventions and static analysis
  • The solution demonstrates the importance of following pointers in reverse engineering
